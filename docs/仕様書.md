# フォト ビューアー 仕様書

## 1. アプリケーション概要

| 項目 | 内容 |
|---|---|
| アプリケーション名 | フォト ビューアー |
| 目的 | ローカルドライブの画像ファイルをフォルダー単位でサムネイル表示する |
| 実行形式 | Windows デスクトップアプリケーション（WinForms） |
| 対象 OS | Windows（.NET 9 以降） |
| エントリーポイント | `Program.Main()` → `Application.Run(new MainForm())` |

---

## 2. 動作環境

| 項目 | 要件 |
|---|---|
| フレームワーク | .NET 9.0-windows |
| UI フレームワーク | Windows Forms |
| 最小ウィンドウサイズ | 700 × 500 px |
| デフォルトウィンドウサイズ | 1100 × 700 px |

---

## 3. 対応画像形式

| 拡張子 | 形式 |
|---|---|
| `.jpg` / `.jpeg` | JPEG |
| `.png` | PNG |
| `.bmp` | BMP |
| `.gif` | GIF |
| `.tiff` / `.tif` | TIFF |
| `.webp` | WebP |

拡張子の大文字・小文字は区別しない。

## 5. 機能仕様

### 5.1 フォルダーツリー

#### 5.1.1 起動時初期化
- 準備済みのすべてのドライブ（`DriveInfo.GetDrives()` で `IsReady == true`）をルートノードとして表示する。
- 各ノードは展開前に「loading...」という仮ノードを持つ（遅延読み込み）。

#### 5.1.2 遅延展開
- ノードを展開すると、対応パスのサブフォルダーを取得してノードとして追加する。
- フォルダーはアルファベット順に並べる。
- `UnauthorizedAccessException`・`IOException` は無視する（ノードを空にする）。

#### 5.1.3 フォルダー選択
- ノード選択時にサムネイルを非同期で読み込む。
- 読み込み中に別フォルダーを選択した場合、前の読み込みをキャンセルして新しい読み込みを開始する。

### 5.2 サムネイル表示

#### 5.2.1 読み込みフロー

1. バックグラウンドスレッドでフォルダー内の画像ファイルを列挙し、ファイル名昇順に並べる。
2. 50 件単位でパネルを UI スレッドに追加し（バッチ処理）、`Task.Yield()` で UI を解放する。
3. 最大 `min(CPU コア数, 8)` の並列度で各画像のサムネイルを並列読み込みする。
4. サムネイルが完成次第、対応パネルへ逐次反映する。

#### 5.2.2 サムネイルパネル（ThumbnailPanel）

各画像ファイルに対して 1 つのパネルを生成する。

| 項目 | 値 |
|---|---|
| サムネイルサイズ | 最大 150 × 150 px（アスペクト比を保持して内接） |
| パディング | 8 px |
| ファイル名表示 | パネル下部にラベル表示（7.5pt Segoe UI、長い場合は省略記号で丸める） |
| 通常背景色 | 白（`#FFFFFF`） |
| ホバー背景色 | 薄青（`#DCEBFC`） |
| 選択背景色 | Windows アクセントブルー（`#0078D7`） |
| カーソル | Hand |

#### 5.2.3 サムネイル生成
- GDI+ の `GetThumbnailImage()` を使用する。JPEG ファイルに EXIF 埋め込みサムネイルが存在する場合は自動的に活用され、高速に処理される。
- サイズは元画像のアスペクト比を維持し、150 × 150 に内接させる。

#### 5.2.4 ステータス表示
- 読み込み中：`N / M 件読み込み済み`（20 件ごとに更新）
- 読み込み完了：`N 件の画像`
- ファイルなし：`画像ファイルがありません`
- アクセス拒否：`アクセスが拒否されました`

### 5.3 画像操作

#### 5.3.1 単クリック（選択）
- クリックしたパネルを選択状態（青背景・白テキスト）にする。
- 以前の選択を解除する。
- ステータスバーにファイル名を表示する。

#### 5.3.2 ダブルクリック（外部アプリで開く）
- OS に関連付けられたアプリケーションで画像ファイルを開く（`ShellExecute`）。

## 6. 設定の永続化

### 6.1 設定ファイル

| 項目 | 値 |
|---|---|
| ファイルパス | `%LOCALAPPDATA%\PhotoViewer\settings.json` |
| フォーマット | JSON（インデント付き） |
| 保存タイミング | アプリケーション終了時 |
| 読み込みタイミング | 起動時 |

### 6.2 保存項目

| プロパティ | 型 | デフォルト値 | 説明 |
|---|---|---|---|
| `WindowWidth` | int | 1100 | ウィンドウ幅（px） |
| `WindowHeight` | int | 700 | ウィンドウ高さ（px） |
| `WindowLeft` | int | -1 | ウィンドウ左位置（-1 は未設定） |
| `WindowTop` | int | -1 | ウィンドウ上位置（-1 は未設定） |
| `WindowState` | int | Normal(1) | ウィンドウ状態（`FormWindowState` の整数値） |
| `SplitterDistance` | int | 200 | スプリッター位置（px） |
| `SelectedFolder` | string | `""` | 最後に選択したフォルダーパス |
### 6.3 起動時の状態復元
- 前回終了時に選択していたフォルダーをツリーで再選択し、サムネイルを自動表示する。
- ウィンドウ位置・サイズ・スプリッター位置を復元する。
- 保存位置がいずれのモニターの作業領域にも重ならない場合は位置復元をスキップする。

## 7. 非機能仕様

### 7.1 UI 応答性
- フォルダー切替時：バッチ処理と `Task.Yield()` により UI スレッドをブロックしない。
- サムネイル読み込み中：`Parallel.ForEachAsync`（最大 8 並列）により並列読み込みを行う。
- フォルダー切替操作：前の読み込み処理を `CancellationToken` で即座にキャンセルする。
- アプリ終了：`Environment.Exit(0)` による即時プロセス終了で、大量の HWND 破棄による応答なしを回避する。

### 7.2 エラー処理

| 状況 | 対応 |
|---|---|
| アクセス権限なし | ステータスバーに「アクセスが拒否されました」を表示し処理を中断 |
| I/O エラー | ステータスバーにエラーメッセージを表示し処理を中断 |
| 画像読み込み失敗 | 該当ファイルをスキップ（サムネイル未表示のまま） |
| 設定ファイル読み込み失敗 | デフォルト値で起動 |
| 設定ファイル保存失敗 | 無視（次回起動時にデフォルト値を使用） |
